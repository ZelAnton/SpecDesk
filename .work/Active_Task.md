## Критерии выполнения
- В `md-blocks.ts` ведущий контент (пустые строки и/или ведущие reference-definitions
  до первого top-level токена) больше не образует отдельный синтетический блок:
  для документа хотя бы с одним блоком `splitTopLevelBlocks(md).length` равно числу
  top-level токенов, то есть `blocks.length === originalDoc.childCount`. Инвариант
  `joinBlocks(splitTopLevelBlocks(md)) === md` сохранён для всех существующих и новых
  фикстур (включая пустой/whitespace-only документ).
- `serializeWithSplice(original, edited)` для документа с ведущей пустой строкой или
  ведущим reference-definition при отсутствии реальных правок возвращает `original`
  байт-в-байт (больше не срабатывает whole-document fallback по несовпадению
  `childCount !== blocks.length`).
- При правке первого блока такого документа его ведущий контент (пустые строки /
  reference-definitions перед нодой) сохраняется дословно — симметрично уже
  существующему сохранению хвоста (`contentLineEnd`/`tailLines`).
- Переключение "Форматированный" → "Код" (`index.ts` applyMode) для неизменённого
  документа с ведущей пустой строкой не переразбирает и не меняет текст: `formatted.getText()`
  на незатронутом документе байт-идентичен исходнику.
- В `formatted.ts` маппинг `blocks[i]` ↔ `doc.child(i)` для документов с ведущим
  контентом совпадает 1:1: подсветка активного блока, hover и scroll-sync указывают на
  правильную ноду (нет смещения на один блок).
- Добавлены фикстуры и тесты: документ с ведущей пустой строкой и документ с ведущим
  reference-definition, прогнанные через md-blocks (round-trip + число блоков),
  md-splice (no-op байт-идентичность + правка первого блока сохраняет ведущий контент)
  и formatted (маппинг подсветки/строк + getText неизменённого документа).
- Запись в `CHANGELOG.md` под `## [Unreleased]`.
- `dotnet build`/тесты .NET не затрагиваются; в `webview/` проходят
  `npm run typecheck`, `npm run lint`, `npm test`.

## Описание
`webview/src/md-blocks.ts:91-94` добавляет синтетический блок в начало (unshift), когда первый токен
начинается не с 0-й строки, поэтому `blocks.length = childCount + 1` навсегда, и `md-splice.ts:52`
берёт полный откат на весь документ при каждом `getText()`. Последствия: (a) одно нажатие клавиши
переписывает весь файл (переразбивка hard-wrap, перенумерация списков, setext→ATX, выравнивание
таблиц, плюс удаление reference-definitions из S-12); (b) `index.ts:399-403` при выходе из режима
"Форматированный" читает `formatted.getText()` и перезаписывает исходный редактор переразобранным
текстом — автор видит изменённый документ без единой правки; (c) `formatted.ts:395-404` индексирует
`blocks[i]` параллельно детям ProseMirror, поэтому подсветки/hover/scroll-sync смещены на один блок.
Каждый тестовый фикстур в проекте удобно начинается с заголовка, поэтому баг ничем не покрыт.

Критерии готовности:
- Синтетический ведущий блок становится реальной, адресуемой нодой (либо смещение блок↔дитя-маппинга
  сохраняет соответствие 1:1) — `blocks.length` остаётся синхронным с `childCount`.
- Документ с ведущей пустой строкой (или ведущим reference-definition) больше не вызывает полный откат
  на каждый `getText()` при отсутствии реальных изменений.
- Переключение "Форматированный" → "Код" больше не переразбирает/не меняет документ, который не
  редактировался.
- Подсветки/hover/scroll-sync корректно совпадают для документов с ведущей пустой строкой.
- Добавлены фикстуры: документ с ведущей пустой строкой и документ с ведущим reference-definition,
  прогнанные через тесты md-blocks/md-splice/formatted.
- Запись в `CHANGELOG.md`.
