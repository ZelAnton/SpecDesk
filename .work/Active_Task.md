# Активная задача

Статус: в работе
Исходная задача: [T-011] Исправить S-12: откатной полный ресериализатор удаляет link-reference-definitions

Оценка сложности: средняя
Рекомендуемый effort: high

`webview/src/md-splice.ts:44-54`. При несовпадении числа блоков откат — это
`serializer.serialize(edited)`; у reference-definitions нет узла ProseMirror (markdown-it резолвит их
на этапе парсинга), поэтому вывод теряет каждую строку `[id]: url`, использованные reference-ссылки
превращаются в inline, а неиспользуемые определения удаляются. §5 документирует, что добавление/удаление
блока приводит к полному ре-сериализации — но не разрешает потерю содержимого. Триггер тривиален:
нажать Enter в WYSIWYG, чтобы начать новый абзац в любом документе с reference-definition → определения
исчезают из сохранённого Markdown.

Критерии готовности:
- Откатной путь при несовпадении числа блоков сохраняет reference-definitions (`[id]: url`) дословно —
  либо переносит их через откат, либо отказывается от отката при их наличии в исходнике (используя
  более безопасный путь).
- Нажатие Enter в WYSIWYG для нового абзаца в документе с reference-definitions больше не удаляет их из
  сохранённого Markdown.
- Добавлен тест "откат сохраняет содержимое" с явной проверкой на reference-definitions (текущий тест
  отката ничего не проверяет по сохранности).
- Запись в `CHANGELOG.md`.
