### [T-068] Унифицировать сравнение идентичности путей в Host (Ordinal vs OrdinalIgnoreCase vs канонический handle-путь) — статус: не начата
В Host сосуществуют три расходящиеся трактовки «это один и тот же путь», и они смешиваются в одних и тех же сценариях:

3. канонические handle-пути удаления (`src/SpecDesk.Host/WindowsHandleFileDeletion.cs`: `AreSameCanonicalHandlePath`, `IsCanonicalHandleDescendant`) — кейс-чувствительный хвост + кейс-нечувствительный volume authority (осознанно, под NTFS case-sensitive каталоги).

Отдельная точечная несогласованность: `IsWorkspaceTreeRequestCurrentLocked` (`HostController.Session.cs`, ~строка 830) сравнивает `publication.DocumentPath` с `_currentPath` через `OrdinalIgnoreCase`, тогда как все соседние проверки того же поля — `Ordinal`.

Это даёт трудноотлаживаемые отказы вида «The open document changed. Open the file menu again and retry» или ложные совпадения активного документа.

Нужно свести к единой, явно задокументированной политике идентичности пути (возможно, две осознанные: «идентичность записи каталога» для деструктивных операций и «идентичность документа» для сессии) и прогнать все перечисленные места через общие хелперы вместо рассыпанных `string.Equals` с разными комparer'ами.

Критерии готовности:

- `IsWorkspaceTreeRequestCurrentLocked` не отличается от соседних проверок без документированной причины.
- Существующие тесты проходят; добавлен тест на путь, отличающийся только регистром (открытие + claim + удаление).
